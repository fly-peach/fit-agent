# Phase 06 开发计划：专业化仪表盘与目标进度洞察

## 1. 阶段目标

- 将当前仪表盘升级为“专业数据洞察中心”，避免用户误读指标。
- 在单页中完成：健康画像、异常预警、目标进度与趋势对比。
- 强化“可解释性”：每个关键指标都提供行业标准参考范围。

## 2. 范围（In Scope）

### 2.1 页面范围

- 仪表盘（`/dashboard`）
- 每日数据（`/daily-metrics`）
- 运动计划（`/daily-workout`）
- 成长分析（`/growth-analytics`）

### 2.2 核心能力范围

- 健康画像卡片（BMI、体脂率、肌肉量、BMR、目标达成进度）
- 行业标准参考值展示（按性别/年龄段可扩展）
- 红黄蓝三级预警标识
- 训练目标进度看板（环形进度图 + 折线趋势图）

## 3. 仪表盘信息架构（目标形态）

### 3.1 核心数据概览区

- 模块 A：健康画像卡片
  - 指标：`BMI`、`体脂率`、`肌肉量`、`基础代谢(BMR)`、`目标达成率`
  - 每个指标包含：
    - 当前值
    - 同期变化（较上周/上月）
    - 行业标准参考范围说明
    - 预警状态（正常/关注/风险）

- 模块 B：训练目标进度看板
  - 目标类型：减脂 / 增肌 / 体能提升
  - 组合图例：
    - 外环：本周关键变化（例：体脂率变化）
    - 内环：当前目标达成率
    - 折线：近 4 周趋势

### 3.2 页面布局建议

- 顶部：健康画像卡片网格（2x3 或 1x5 响应式）
- 中部：目标进度复合图（环形 + 趋势）
- 底部：预警列表与行动建议（本周重点）

## 4. 专业规则定义

### 4.1 行业参考值（首版默认口径）

- 体脂率（男性）：健康参考 `10% - 20%`
- BMI：健康参考 `18.5 - 23.9`
- 静息心率：正常参考 `60 - 100 bpm`
- BMR：按身高体重年龄估算后给出“低/正常/高”区间标签

> 说明：首版可先用固定参考区间，后续扩展为按性别、年龄、训练阶段动态范围。

### 4.2 三级预警规则（红黄蓝）

- 红色（高风险）：明显偏离健康区间或存在急性风险信号
  - 示例：静息心率 `> 100 bpm`
- 黄色（中风险）：轻中度偏离区间，建议调整训练/饮食
- 蓝色（提示）：接近边界值，建议持续观察

## 5. 前后端改造点

### 5.1 后端接口与数据

- 保持主入口：`GET /api/v1/dashboard/me`
- 新增/扩展字段（建议在 `growth_analytics` 下）：
  - `health_profile`: 关键指标 + 参考范围 + 预警等级
  - `goal_progress`: 外环值、内环达成率、近4周趋势点
  - `alerts`: 预警列表（level、metric、message、action）

### 5.2 前端展示与交互

- 仪表盘新增组件：
  - `HealthProfileCards`
  - `GoalProgressBoard`
  - `RiskAlertList`
- 每张卡片显示“参考范围提示文案”（非 hover 隐藏，需默认可见）
- 颜色语义统一：
  - 红：风险
  - 黄：关注
  - 蓝：提示

## 6. 与每日记录页面联动要求

- `/daily-metrics` 录入后应直接影响健康画像卡片与目标达成率。
- `/daily-workout` 录入后应影响“体能提升”目标趋势。
- `/daily-nutrition`（通过成长分析页使用）应影响 BMR 解释与建议提示（首版可先提供基础关联提示）。

## 7. 验收标准（Definition of Done）

- 仪表盘出现“健康画像卡片 + 目标进度看板 + 预警区”三大模块。
- 关键指标均展示行业参考范围文案，且可被用户直接看到。
- 异常值触发红黄蓝三级预警且文案正确。
- 目标看板同时显示“当前进度 + 趋势变化”。
- 从每日记录录入到仪表盘可视化的链路可验证（数据一致）。

## 8. 风险与注意项

- 参考值口径需在文档中明确来源，避免专业争议。
- 首版图表以“可读、稳定、可解释”为优先，复杂动画后置。
- 若用户基础信息缺失（性别/年龄），需使用默认口径并提示“参考值为通用范围”。
