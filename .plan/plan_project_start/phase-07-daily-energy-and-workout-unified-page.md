# Phase 07 开发计划：热量摄入与运动计划一体化页面

## 1. 阶段目标

- 将“每日热量摄入”和“每日运动计划”从两个独立页面合并为一个统一工作台。
- 让用户在同一页面完成“计划 -> 摄入 -> 执行 -> 复盘”的日常闭环。
- 提升操作效率，减少跨页面跳转导致的数据割裂。

## 2. 范围（In Scope）

### 2.1 页面与路由

- 新增统一页面：`/daily-energy-workout`
- 原路由兼容策略：
  - `/daily-workout` 与 `/daily-nutrition` 可保留短期重定向到新页面
  - 后续版本再评估是否下线旧入口

### 2.2 数据能力

- 继续复用现有后端接口：
  - `PUT/GET /api/v1/daily-workout`
  - `PUT/GET /api/v1/daily-nutrition`
- 页面级联动展示：
  - 当日训练计划摘要
  - 当日热量与三大营养素摄入
  - “摄入 vs 计划消耗”差值提示（首版可采用估算）

## 3. 页面结构（详细设计）

## 3.1 整体布局

- 采用 `左主编排 + 右侧摘要` 的双栏布局（移动端改为上下布局）
- 顶部固定“日期切换条”，全页数据按同一天同步切换

### A 区：顶部日期与状态条
- 元素：
  - 日期选择器（默认今天）
  - 今日状态标签：未完成 / 进行中 / 已完成
  - 一键保存按钮（保存训练与营养）
- 规则：
  - 切换日期后，训练与营养表单同时回填该日期数据

### B 区：左侧主内容（编辑区）

#### B1 模块：每日运动计划
- 字段：
  - 计划标题
  - 动作清单（动作名、组数、次数、时长）
  - 总时长（分钟）
  - 完成状态（checkbox/switch）
  - 备注
- 交互：
  - 支持动作行新增/删除
  - 支持动作模板快速填充（首版可预置 3 套）
  - 保存时采用当日 Upsert

#### B2 模块：每日热量摄入
- 字段：
  - 总热量（kcal）
  - 蛋白质（g）
  - 碳水（g）
  - 脂肪（g）
  - 备注
- 交互：
  - 输入实时计算营养供能估算（蛋白/碳水*4，脂肪*9）
  - 校验非负数，允许 0
  - 保存时采用当日 Upsert

### C 区：右侧信息摘要（分析区）

#### C1 今日执行摘要卡
- 展示：
  - 训练总时长
  - 训练完成状态
  - 热量总摄入
  - 三大营养素比例（蛋白/碳水/脂肪）

#### C2 差值提示卡（关键）
- 展示：
  - 摄入热量 - 目标消耗（估算）= 差值
  - 差值状态：偏低 / 合理 / 偏高
- 文案规则（首版）：
  - 差值 > +300：提示“摄入偏高，建议控制晚餐或加餐”
  - 差值 < -300：提示“摄入偏低，注意恢复与训练质量”
  - 其他：提示“摄入与计划匹配度良好”

#### C3 最近7天微趋势
- 图表：
  - 热量折线
  - 训练时长柱状
- 目的：
  - 让用户看到短周期行为一致性

## 4. 组件拆分建议（前端）

- `DailyEnergyWorkoutPage` 页面容器
- `DateToolbar` 日期与状态条
- `WorkoutPlanEditor` 训练计划编辑器
- `NutritionEditor` 营养摄入编辑器
- `TodaySummaryCard` 今日摘要卡
- `EnergyGapCard` 差值提示卡
- `WeekMiniTrend` 最近7天微趋势图

## 5. API 调用与数据流

1. 页面初始化：
- 并发请求当日 workout + nutrition
- 并发请求最近7天列表用于趋势图

2. 页面保存：
- 点击“保存全部”时并发提交：
  - `PUT /daily-workout/{record_date}`
  - `PUT /daily-nutrition/{record_date}`
- 任一失败时展示明确错误并保留本地输入

3. 日期切换：
- 触发双模块回填 + 右侧摘要重算

## 6. 校验与异常处理

- 输入校验：
  - 时长、热量、营养素、组数、次数必须为 `>=0`
  - 动作名不能为空
- 异常处理：
  - 保留后端 detail 原文，展示在对应模块顶部
  - 网络失败支持重试

## 7. UI 设计规范（本阶段）

- 色彩：延续浅蓝 + 白主视觉
- 层级：编辑区强调操作，摘要区强调结果
- 间距：模块上下间距 16-20px，卡片内边距 16px+
- 可用性：
  - 一屏可完成核心操作（保存训练 + 营养）
  - 按钮文案统一使用动词开头（保存、新增、删除）

## 8. 验收标准（Definition of Done）

- 用户可在一个页面完成当日训练计划与热量摄入的录入和保存。
- 切换日期时两个模块同步回填，不出现跨日数据混淆。
- 右侧摘要区实时反映输入变化与差值提示。
- 最近7天趋势可正常展示（热量 + 训练时长）。
- 页面构建通过，核心交互无阻断错误。

## 9. 风险与注意项

- 训练“消耗估算”口径首版为简化算法，需在文案中标注“估算值”。
- 同时保存双模块时要保证失败可回滚提示（至少前端提示明确）。
- 移动端需保证输入区域不拥挤，必要时折叠右侧摘要卡。
